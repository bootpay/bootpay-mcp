왜 하나로 합쳤는가

Stripe는 결제·고객·구독·인보이스·웹훅 등 30개 이상의 도구를 단일 MCP(@stripe/mcp)로 제공한다. Bootpay도 같은 원칙을 따랐다.

Stripe @stripe/mcp
  • 결제·고객·구독·인보이스 등 30+ 도구를 하나의 MCP로
  • Restricted API Key 권한으로 노출 도구 필터링
  • Remote (mcp.stripe.com) + Local (npx) 듀얼 전송
  • npx가 항상 최신 버전 fetch — 자동 업데이트
Bootpay @bootpay/mcp
  • Docs + Admin + Commerce 54개 도구를 하나의 MCP로
  • 전송 방식(HTTP/stdio)과 인증 레벨로 도구 범위 제어
  • Remote (mcp.bootpay.ai) + Local (npx) 듀얼 전송
  • Docs는 Cloudflare KV 동기화, 나머지는 npx 자동 업데이트

Before — 3개 MCP

❌ 사용자가 MCP 3개를 각각 설정
❌ bootpay-mcp는 HTTP only, admin-mcp는 stdio only
❌ 인증 체계 분리 (없음 / JWT / Basic Auth)
❌ 업데이트 경로 불일치 (Cloudflare / npm / 미정)
→

After — 1개 MCP

✓ npx @bootpay/mcp 하나로 끝
✓ HTTP + stdio 듀얼 전송 지원
✓ 하나의 서버에서 3종 인증 (Public / JWT / Basic Auth)
✓ npm + wrangler 단일 소스
